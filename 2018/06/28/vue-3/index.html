组件

  • 允许我们使用小型、独立、通常可复用的组件构建大型应用，方便调错

  • 全局组件：

    • Vue.component(‘组件名’,{模板内容})
    • 组件内容只能有一个根元素
    • 组件名声明时不要使用驼峰命名，可使用Kxxxxxx(首字母大写)或者xxx-xxx,和普通的DOM节点进行区分
    • 注册全局化组件也要实例化Vue,组件可以在DOM中调用多次

  • 组件下的data必须是函数模型，一个组件下的data必须是一个函数,因此每个实例可以维护一份被返回对相关的独立拷贝

  • 局部组件：

    • 在Vue实例化下,component:{组件名:{‘模板’},组件名:{‘模板’}},其他内容和全局组件一样

  • 使用DOM嵌套标签时，组件封装后须使用is插入到DOM中

  • 组件的作用和优点，什么组件化开发

    • 作用和优点：
      • 提高开发效率
      • 方便重复使用
      • 简化调试步骤
      • 提升整个项目的可维护性
      • 便于协同开发
    • 组件化开发：
      • 组件可以扩展 HTML 元素，封装可重用的代码

props传值

  • 可以把组件复用中变动的值提取出来使用props进行传值,提高组件的复用率 n=可以当成函数中的实参
    props:[‘可以当成函数的形参’,’name’],
  • 使用props传值如果实际传值的位置使用横杠，接收porps值的位置要使用驼峰命名规则，不建议使用 my-msg=”2” 接收时props:[‘myMsg’]
  • 动态传递props：<my-component v-bind:props值=”data下的数据”/>，父级中的数据向子级中去传递时，使用动态绑定props的方法进行传递，使用动态传递props值时可以传递多种数据类型，提高组件复用率
  • 什么时候要用poros验证：当协同开发时，开发好的组件别人在使用时，只是用模板，不使用数据，数据一般情况下会自己定义，但是很多情况下别人不知道该传入什么数据类型，在这种情况下要使用props验证

条件渲染

  • v-if v-else-if v-else 可以配合使用，配合使用时中不要插入其他DOM节点
  • template 可以使用模板渲染多条内容,且不需要渲染根节点时使用
  • v-if=”可以使用表达式”
  • 使用v-if切换节点时，vue默认是高效渲染的，就地复用元素，Vue 会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染。 这么做除了使 Vue 变得非常快之外，还有其它一些好处,当加入唯一的key值时可以让DOM节点摧毁重建,V-if和v-show的区别:v-show改变style中的样式,v-if把整个节点摧毁重建,一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。
