  • vue-resource老版本请求数据的模式，现在被vue-axios替代

    • get请求本地数据时，把本地文件放在static文件夹下
    • 在main.js下引入import Resource from ‘vue-resource’
      使用use的方法Vue.use(Resource) node中的中间件
    • this.$http.get(路径).then((数据)=>{请求成功的数据})
    • 如果请求后台接口，需要在config文件下的index解决跨域问题，找到proxyTable属性配置代理
    • this.$http.get(代理接口).then((数据)=>{请求成功的数据})
    • Post:this.$http.post(‘路径接口’,{传到后台的数据})
      .then(()=>{成功}).catch(()=>{失败})
      如果成功可以判断成功的状态给用户提示，失败给用户提示

  • Axios

    • main.js下引入axios :import axios from ‘axios’;
    • main.js下把axios写入Vue原型：Vue.prototype.axios = axios;
    • 配置代理
    • this.axios.get(‘路径’).then((res)=>{成功}).catch(()=>{失败})
    • this.axios.post(‘路径’,{对象传入后台的数据}).then((res)=>{成功}).catch(()=>{失败})
    • axios请求别名，和H5语义化相同，在语境下去使用，一般可以使用get和post请求去代替
      • axios.request(config)
      • axios.get(url[, config])
      • axios.delete(url[, config])
      • axios.head(url[, config])
      • axios.options(url[, config])
      • axios.post(url[, data[, config]])
      • axios.put(url[, data[, config]])
      • axios.patch(url[, data[, config]])

  • Vuex状态管理

  • 把公用的数据抽离出来进行统一的管理。在大型项目下去使用，小项目中去使用反而让项目更加繁杂。

    • 在src文件夹下写入一个store的文件夹,在store下创建store.js
    • 在store.js下引入Vue和Vuex并用use方法使用Vuex
    • 创建store const store = new Vuex.Store({}) 并把store暴露出去
    • 在main.js下引入store,在Vue实例化中挂在store
    • 创建公用数据仓库state,把公用数据写在state下
    • 在组件内容使用数据this.$store.state.数据
    • getters,store下公用数据的计算属性，可以使用方法把state下边的值计算好之后返回出去，在组件内使用this.$store.getters.方法
    • mutations:store下的公用的methods，可以触发事件改变state下的值，在mutations下可以直接写入方法后不能够直接触发，需要在组件中使用commit的方法去触发this.$store.commit(‘字符串store,mutations下的事件名’)
    • Actions是异步的mutations，在actions写入方法之后去触发mutations下边的方法 store.commit(mutations下的方法’,参数);，在组件中this.$store.dispatch(‘mutations下的方法’,参数) dispatch分发
    • Modules可以把store下边的内容拆分成一个一个的模块然后再去使用，使用方法和store一直
